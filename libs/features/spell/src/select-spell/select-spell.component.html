HEADS UP: this is how the spell_dbc table works

The spell_dbc table contains server-side-only spell entries. It has been cleaned to no longer contain client-side spell data. Patches for client-side spells are done via SpellInfoCorrections. For the up-to-date table documentation see spell_dbc.

IMPORTANT: Although spell_dbc entries are loaded after the client DBC (load order: spell.dbc -> spell_dbc -> SpellInfoCorrections), it's not recommended to use spell_dbc to overwrite client-side spell data. If you do want to add new overrides, follow THIS HOW-TO.
